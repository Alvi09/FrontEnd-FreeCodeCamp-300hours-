Introduction

JavaScript is a cross-platform, object-orientated scripting language. It's small and lightweight. Inside say, a web browser (or any host environment), JavaScript can be connected to objects of its environment, which'll provide programmatic control over them.

JavaScript contains a standard library of objects, like Array, Date, and Math, and a core set of language elements such as operators, control structures, and statements. Core JavaScript can be extended for a variety of purposes:

  • Client-side JavaScript extends core language by supplying objects to control a browser and its Document Object Model (DOM). For example, you could allow applications to place elements on an HTML form and respond to user events such as mouse clicks, form input, and page navigation.
  • Server-side JavaScript extends core language by supplying objects relevant to running JavaScript on a server. For example, you could allow applications to communicate with a database, which provides continuity of informatiion from one inovcation to another, or perform file manipulations on a server.

JavaScript and Java

What's the difference between these two? Well, first of all, JavaScript doesn't have Java's static typing nor strong type checking, but it does folow Java's expression syntax, naming conventions, and basic control-flow constructs.

In constrast to Java's compile-time system of classes built by declarations, JavaScript supports a runtime system based on a small number of data types representing numeric, boolean, and string values. Unlike the more common class-based object model, JavaScript is prototype-based; This'll provide dynamic heritance. JavaScript also supports functions without special declarative requirements. Functions can be properties of objects, executing as loosely typed methods.

JavaScript is also an incredibly free-form language compared to Java. There's no need to declare all variables, classes, and methods, nor is there a need to worry about public, private, or protected methods. We also don't have to implement interfaces. Also, variables, parameters, and function return times aren't explicitly typed.

Variables

As in other languages, variable names, often called identifiers , conform to certain rules.

It must start with a letter, underscore (i.e _) or dollar sign (i.e $) Characters following can be digits (i.e 0-9). JavaScript is also case sensitive.

We can use ISO 8859-1 or Unicode letters such as Ã¥ in identifiers. Some legal names are Number_hits, temp99, and _name.

Declaring variables

We can declare a variable in three ways:

With the keyword var. For example, var x = 42 This can be used to declare both local and global variables.

By assigning it to a value. For example, x = 42 This always declare a global variable and generates a strict JavaScript warning; You shouldn't use this variant.

With the keyword let. For example, let y = 13 This syntax can be used to declare a block scope local variable. See below for more info about variable scope.

Variable scope

Global functions are when you declare a variable outside a function, whereas local variables are declared inside the function.

Before ECMAScript 2015, JavaScript didn't have any block statement scopes, but instead had a variable declared within a block, which is considered local to the function (or global scope). For example, the following code will log 5 because the scope of x is the within the function which x was declared, not the block, which is an if-statement in this case. if (true) { var = 5; } console.log(x); // 5

The behavior changes when using the let declaration introduced in ECMAScript 2015. if (true) {let y = 5; } console.log(y) // ReferenceError: y is not defined

Global variables

Global variables are properties of the global object. For example, in web pages the global object is the window; We can set and access global variables using the window-variable syntax.

Consequently, we can access global variables declared in one window or frame from another window by specifiying the window or frame name. For example, if we have a variable called phoneNumber and it's declared in a document, we can refer to this variable from an iframe as parent phoneNumber.

Constants

We can create read-only, named constants with the const keyword. Syntax follows the same naming rules as variable identifier. const PI = 3.14;

A constant cannot be changed through assignment or re-declared while the script is running. It also has to be initialized to a value.

Scope rules for constants are same as let block scope variables. If the const keyword is omitted, the identifier is assumed to represent a variable.

You cannot declare a constant with the same name as a function or variable in the same scope. For example, // THIS WILL CAUSE AN ERROR funciton f() {}; const f = 5; // THIS WILL ALSO CAUSE AN ERROR function f() { const g = 5; var g;} //statements

However, object attributes aren't protected, so this would execute without errors. const MY_OBJECT = {"key": "value"}; MY_OBJECT.key = "otherValue";

Data types

ECMAScript standard defines seven data types:

  • Six data types that're primitives:
    • Boolean. true and false.
    • null. A special keyword denoting a null value. Note that because JavaScript is case-sensitive, null is not the same as Null, NULL, or any other variant.
    • undefined. A top-level property whose vlaue is undefined.
    • Number. 42 or 3.14159.
    • String. "Howdy"
    • Symbol (new in ECMAScript 2015). A data type whose instances are unique an immutable.
  • and Object

Although these data types are small, they'll allow you to perform useful functions with your applications. Objects and functions are fundamental elements. You can think of objects as named containers for values, and functions as procedures that your application can perform.

if...else statement

Use if statement to execute a statement if a logical condition is true. Use the optional else if the condition is false. if (condition { statement_1; } else { statement_2; }

We can also compound the statements using an else if to have multiple conditions tested. if (condition_1) { statement_1; } else if (condition_2) { statement_2; } else if (condition_n) { statement_n; } else { statement_last; }

In case of multiple conditions, only the first logical condition which evaluates to true is executed. In order to execute multiple statements, we should group thgem within a block statement ({. . . }). And generally, it's good practice to use block statements, especially when nesting if statements. if (condition) { statement_1_runs_if_condition_is_true; statement_2_runs_if_condition_is_true; } else {statement_3_runs_if_condition_is_false; statement_4_runs_if_condition_is_false; }

It's adviseable to not use simple assignments in a conditional expression because assignments can be confused with equality while glancing over code. For example, do NOT use the following code. if (x = y) { /* statement here */ }

If you need to use an assignment in a conditional expression, it's common practice to put an additional parenthesis around the assignment. For example, if ((x = y)) { /* statement here */ }

while statement

A while statement executes its statement as long a specified condition evaluates to true. while (conditional) statement

If condition becomes false, statement within loop stops executing and control passes to the statement following the loop.

The condition test occurs before the statement in the loop is executed. If the condition returns true, the statement is executed and the condition is tested again. If it returns false, execution stops and control is passed to the statement following while.

To execute multiple statements, use a block statement ({ ... }) to group those statements.

For example, the following while loop iterates as long as n is less than three:

var n = 0; var x = 0; while (n < 3) { n++; x +=n; }

With each iteration, the loop increments n and adds that value to x. Therefore, x and n take on the following values:

  • After the first pass: n = 1 and x = 1
  • After the first pass: n = 2 and x = 3
  • After the first pass: n = 3 and x = 6

After completing the first pass, the condition n < 3 is no longer true, so the loop terminates.

Function declarations

Function definition (often called function declaration) consists of the function keyword, follwoed by:

  • The name of the function
  • A list of arguments to the function, enclosed in parentheses and separated by commas
  • JavaScript statement that define the function, enclosed in curly brackets, {}.

For example, the following code defines a simple function named square: function square(number) { return number * number; }

The function square takes one argument, called number. The function consists of one statement that says to return the arg of the function (i.e the number multipled by itself).

Primitive parameters (such as a number) are passed to functions by value; the value is passed to the function, but if the function changes the value of the parameter, this change isn't reflected globally or in the calling function.
